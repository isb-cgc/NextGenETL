cda_bq_etl.bq_helpers.schema.create_and_upload_schema_for_json(params: dict[str, str | dict | int | bool | dict[str, str | dict | int | bool | dict[str, str | int | bool | dict]]], record_list: list[dict[str, None | str | float | int | bool]], table_name: str, include_release: bool = False, release: str | None = None, schema_fp: str | None = None, delete_local: bool = True, reorder_nesting: bool = False)¶

Create a schema object by recursively detecting the object structure and data types, storing result, and converting that to a Schema dict for BQ ingestion.

Parameters:

  • params (Params) – params supplied in yaml config

  • record_list (JSONList) – list of records to analyze (used to determine schema)

  • table_name (str) – table for which the schema is being generated

  • include_release (bool) – if true, includes release in schema file name; defaults to False

  • release (Optional[str]) – custom release value; defaults to None, in which case the value is derived from the yaml config

  • schema_fp (Optional[str]) – path to schema location on local vm, defaults to None, in which case the value is derived from the yaml config

  • delete_local (bool) – delete local file after uploading to cloud bucket

  • reorder_nesting (bool) – whether the data_types_dict should be reordered, defaults to False

Create and upload schema for a file in tsv format.

Parameters:

  • params (Params) – params supplied in yaml config

  • tsv_fp (str) – path to tsv data file, parsed to create schema

  • header_row (Optional[int]) – integer index of header row within the file; defaults to None

  • skip_rows (int) – integer representing number of non-data rows at the start of the file; defaults to 0

  • schema_fp (Optional[str]) – path to schema location on local vm; defaults to None

  • delete_local (bool) – delete local file after uploading to cloud bucket; defaults to True

  • sample_interval (int) – how many rows to skip between column type checks; defaults to 1

cda_bq_etl.bq_helpers.schema.reorder_data_types_dict(params: dict[str, str | dict | int | bool | dict[str, str | dict | int | bool | dict[str, str | int | bool | dict]]], data_types_dict: dict[str, str | dict]) → dict[str, str]¶

Manually repair mis-ordered data type dict. (If a field in the first entity is null, nested values in data_types_dict get reordered.) There’s probably a better way to go about this, like reordering the data_types_dict before it’s returned, but currently only one table in ICDC uses nesting and making changes to recursively_detect_object_structure would impact every pipeline.

Parameters:

  • params (Params) – params supplied in yaml config

  • data_types_dict (dict[str, str | dict]) – a dictionary representing the data structure and field types.

Returns:

reordered data_types_dict

Return type:

dict[str, str]
